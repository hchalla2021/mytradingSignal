## ğŸ¬ User Journey

### First Load
1. **0s**: User clicks on symbol button
2. **0-100ms**: Component fetches API
3. **100-150ms**: API calculating 14 signals
4. **150ms**: Response received, state updated
5. **160ms**: Component renders with data
6. **Visual**: Fade-in animation of signal cards

### Subsequent Updates (Every 5 seconds)
1. **0s**: Auto-fetch triggered
2. **0-10ms**: API returns (from cache)
3. **10-20ms**: State updated
4. **20-30ms**: Component re-renders
5. **30ms**: Confidence bars smoothly update

### User Clicks Different Symbol
1. **0s**: Click event on new symbol button
2. **Instant**: Button visual feedback (highlight)
3. **0-100ms**: Cancel previous fetch, start new fetch
4. **100-150ms**: New data arrives
5. **160ms**: Component smoothly transitions to new symbol
